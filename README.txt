1. В папке backup лежит скрипт backup.sql для создания и заполнения таблицы, используемой в приложении.
В папке docs лежат тестовые образцы файлов.
В папке logs лежит файл с логом DataSyncApp.log. Для изменения расположения и имени лог файла перейти в \src\main\resources, и отредактировать log4j2.properties:

# имя и папка лог файла в проекте
property.filename = logs/DataSyncApp.log

2. На машине должна быть установлена JDK не ниже версии 1.8.0_144, и настроена переменная среды JAVA_HOME, указывающая на папку установки.

3. Для создания и заполнения таблицы с данными необходимо в СУБД Oracle выполнить скрипт backup.sql.

4. Для подключения к БД необходимо перейти в папку src\main\resources, и в файле dbconnection.properties выполнить настройку подключения.
Для этого в строке подключения к БД нужно задать имя/ip-адрес хоста, порт и SID БД, ниже поменять имя пользователя и пароль:

# строка подключения к БД
dbUrl = jdbc:oracle:thin:@DESKTOP-U5AI89T:1521:xe
# имя пользователя
dbUsername = DZ
# пароль
dbPassword = 123

5. Для запуса открыть cmd, перейти в папку проекта, запустить RunDataSync.bat с указанием через пробел команды и имени файла, например:
RunDataSync.bat sync file.xml
Поддерживаемые команды: export - выгрузка БД в файл, sync - синхронизация БД по файлу.
Файл должен лежать в папке проекта и иметь расширение xml.

При первом запуске будут скачаны необходимые для работы gradle библиотеки.
